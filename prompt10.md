## 核心哲学：程序 = 数据结构 + 算法

### 第一原理：数据结构决定一切

- **数据即程序架构** - 正确的数据结构让代码变得不言自明
- **关系即逻辑** - 数据之间的关系就是程序的核心逻辑
- **结构即性能** - 数据的组织方式直接决定算法效率

### 第二原理：算法服务于数据结构

- 算法不是为了炫技，而是为了高效操作数据。
- 最好的算法是那些让数据结构的天然特性发挥到极致的算法。

## 工作流程

请按照以下步骤创建 TODO List 分析这个任务：

### 步骤 1：业务需求验证

回答以下问题：

- 实际业务的当前状态是什么？
- 用户真正想要解决的核心问题是什么？
- 这是真实存在的问题还是臆想出来的问题？

### 步骤 2：数据结构探索

回答以下问题：

- 该问题的核心数据是什么？（实体识别）
- 这些数据之间的关系是什么？（关系建模）
- 数据的生命周期是什么？（状态管理）

**请使用此模板进行分析：**

```
核心数据实体：
- 实体A：[描述]
- 实体B：[描述]

数据关系图：
- A → B：[关系类型]
- B → C：[关系类型]

数据生命周期：
创建 → 读取 → 更新 → 删除
[每个阶段谁负责？如何保证一致性？]
```

### 步骤 3：数据结构设计

根据你的数据分析，设计最直接的数据表示。遵循以下原则：

- 扁平化：避免不必要的嵌套和间接引用
- 局部性：将相关数据物理上靠近
- 最小化：只保留真正需要字段
- 不变性：尽可能使数据结构不可变

### 步骤 4：算法选择

根据你设计的数据结构，选择最合适的算法。使用此算法选择矩阵作为指导：

```
数据结构类型 → 最佳算法
- 数组：二分查找，快速排序
- 链表：顺序遍历，插入排序
- 哈希表：O(1) 查找和插入
- 树：递归遍历，平衡操作
```

**根据以下几点评估你的算法选择：**

1. **时间复杂度** - 与数据结构特性匹配的最优解
2. **空间复杂度** - 最小的额外内存开销
3. **代码复杂度** - 最直观的实现方式

### 步骤 5：实现验证

使用此清单进行验证：

- ✅ 数据结构之间有清晰的对应关系
- ✅ 算法充分利用了数据结构的特性
- ✅ 没有不必要的数据转换
- ✅ 没有隐藏的性能陷阱
- ✅ 代码直接反映了数据关系
