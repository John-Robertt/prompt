## 第六部分：Agent 提示词架构——从单次提示到系统级构建

随着推理模型的成熟，提示词工程已从单次对话级别的文本优化，扩展到了多轮次、多工具、多 Agent 的系统级架构设计。三大平台在 Agent 系统的设计上展现了各自独特但日趋趋同的架构哲学。

### 6.1 提示件工程的全生命周期

自然语言指令已实质性地成为控制和驱动 AI 系统的代码本身。提示件工程（Promptware Engineering）将提示词视为具有完整生命周期的一等软件工件，要求对其进行版本控制、系统化评估和架构规划。

| 架构维度 | 传统软件运行时 | LLM 提示件运行时 |
| --- | --- | --- |
| 结构与形式化 | 极度结构化，依赖严密语法树 | 基于自然语言，灵活但存在歧义 |
| 执行确定性 | 绝对确定性 | 概率驱动的非确定性 |
| 正确性定义 | 清晰的形式化标准 | 不存在普适的绝对正确边界 |
| 访问控制 | 操作系统权限、内存保护 | 极度脆弱，易受对抗性指令攻击 |

### 6.2 三平台 Agent 模式的跨平台分析

**Anthropic Claude Opus 4.6 的 Agent 架构**：

Claude 4.6 在 Agent 系统上的核心优势是**长 horizon 推理与状态跟踪**。模型"在扩展会话中保持方向感，通过关注增量进展——一次稳步推进几件事而非试图一次完成所有事情"。

关键机制包括：
- **上下文感知**（Context Awareness）：模型能够追踪其剩余上下文窗口，据此规划行为。
- **多窗口工作流**：任务可以跨多个上下文窗口持续推进，模型通过结构化状态文件（JSON）和 git 日志维持记忆连贯性。
- **子 Agent 编排**：Claude 4.6 展现出"显著改善的原生子 Agent 编排能力"，能够主动识别适合委派给子 Agent 的任务。但 Anthropic 也警告过度使用——"Claude Opus 4.6 对子 Agent 有强烈的偏好，可能在简单直接方法足够的场景中生成子 Agent" 。
- **自主性与安全的平衡**：建议添加确认机制——"考虑操作的可逆性和潜在影响。鼓励执行本地可逆操作（如编辑文件、运行测试），但对于难以逆转的操作，在执行前征求用户确认" 。

**OpenAI GPT-5.4 的 Agent 架构**：

GPT-5.4 引入了多项具有架构意义的 Agent 原语：
- **Responses API 与 CoT 传递**：通过 `previous_response_id` 将前一轮次的推理链传递给模型，"减少了推理 Token 的生成、提高了缓存命中率、降低了延迟" 。
- **tool_search 延迟加载**：通过延迟工具定义的加载，"减少 Token 使用量并改善大型工具生态中的工具选择准确性" 。
- **phase 字段**：使用 `phase: "commentary"` 标记中间更新、`phase: "final_answer"` 标记完成的响应，防止中间推理被误解为最终答案 。
- **Computer Use**：GPT-5.4 是"第一个内置计算机使用能力的主线模型"，能够直接与软件界面交互完成任务。
- **Native Compaction**：GPT-5.4 是"第一个经过训练以支持压缩的主线模型"，使更长的 Agent 轨迹成为可能。

GPT-4.1 时代确立的 **Agent 三要素**（persistence + tool-calling + planning）在 GPT-5.4 上依然有效，但实施方式已经演进——推理模型自身已经内化了规划能力，不再需要通过提示词显式诱导。

**Google Gemini 3.1 Pro 的 Agent 架构**：

Gemini 的 Agent 框架提供了最详尽的**行为维度分类**，涵盖三大类九个维度：

1. **推理与策略**：逻辑分解、问题诊断、信息穷尽性
2. **执行与可靠性**：适应性、持久性与恢复、风险评估
3. **交互与输出**：歧义与权限处理、冗余度、精确性与完整性

Gemini 还提供了一个结构化的 Agent 系统指令模板，将上述九个维度组织为"逻辑依赖与约束 → 风险评估 → 溯因推理 → 结果评估 → 信息可用性 → 精确性 → 完整性 → 持久性 → 响应抑制"的优先级链条。

### 6.3 Agent 系统中 UPGA 的扩展

在多 Agent 系统中，UPGA 的五层架构通过**层继承与层特化**机制进行扩展：

- **编排层**（Orchestrator）：拥有完整的五层 UPGA，定义全局目标、上下文和约束。
- **子 Agent**：继承编排层的层 1（意图声明）和层 2（上下文架构），但拥有自己特化的层 3（行为规约）和层 4（推理委托）。
- **验证层**：在编排层面统一管理，确保所有子 Agent 的输出满足全局验证标准。
