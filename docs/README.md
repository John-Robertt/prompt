# 推理模型时代的提示词应用统一理论

基于 GPT-5.4、Claude Opus 4.6 与 Gemini 3.1 Pro 的系统化生成与优化方法论

---

## 核心论点

> **逆复杂度原则**：模型的内置推理能力越强，最优提示词的结构复杂度越低。

推理模型时代（2025-2026），三大平台同步收敛到同一结论：从"教模型如何思考"转向"定义目标与约束，让模型自行决定如何思考"。

## 统一提示词生成架构（UPGA）速览

```
[层2: 上下文架构]  ← 静态参考内容前置
[层1: 意图声明]    ← 目标、成功标准、范围
[层3: 行为规约]    ← 格式约束、正向行为指令
[层4: 推理委托]    ← API参数控制推理深度（非提示词文本）
[层5: 验证迭代]    ← 验证标准后置
```

## 三平台推理参数速查

| 平台 | 参数 | 级别 | 默认 |
| --- | --- | --- | --- |
| OpenAI GPT-5.4 | `reasoning_effort` | none / low / medium / high / xhigh | none |
| Anthropic Claude 4.6 | `effort` (adaptive thinking) | low / medium / high / max | 自适应 |
| Google Gemini 3.1 Pro | `thinking_level` | LOW / MEDIUM / HIGH | HIGH |
