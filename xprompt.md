## 核心原则

IMPORTANT: 数据结构决定程序架构。正确的数据结构让算法选择变得显而易见。
IMPORTANT: 算法必须充分利用数据结构的天然特性，避免与数据组织方式对抗。

## 工作流程

使用 TodoWrite 工具管理以下三步流程：

### 1. 数据建模

识别核心数据实体和关系：

- 核心数据是什么？
- 数据间的关系类型（1:1, 1:N, M:N）
- 数据的生命周期（创建→读取→更新→删除）

### 2. 操作分析

分析数据使用模式：

- 最高频操作是什么？（查找/插入/更新/删除）
- 读写比例
- 访问模式（随机访问 vs 顺序遍历）
- 性能要求（时间/空间约束）

### 3. 结构选择

基于操作频率选择数据结构：

- 数组：随机访问，固定大小
- 哈希表：O(1)查找，无序数据
- 树结构：有序数据，范围查询
- 图结构：复杂关系建模

user: 需要实现一个用户权限管理系统
assistant: 使用 TodoWrite 创建任务：
1. 分析权限系统的核心数据
2. 识别权限检查的操作模式
3. 选择合适的数据结构

核心数据实体：

- 用户 (User)
- 角色 (Role)
- 权限 (Permission)

数据关系：

- User ←→ Role: M:N (用户可有多角色)
- Role ←→ Permission: M:N (角色可有多权限)

最高频操作：权限检查 (hasPermission)
推荐结构：用户ID → Set<权限ID> 的哈希映射
理由：权限检查是 O(1) 操作，空间换时间
</example>
